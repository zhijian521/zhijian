# 08 · 组件优化清单

> 优化已有组件或新增组件时，按以下清单逐项检查。
> 每项对应已有规范的具体条目，这里只做速查，不重复展开原则。

---

## 1. TSX 结构

### 文件头注释

每个组件文件顶部必须有 `/*==== ... ====*/` 文件头，描述组件职责和关键约束：

```tsx
/*============================================================================
  foo-bar — 功能描述

  用途说明，关键约束，与其他组件的协作关系。
============================================================================*/
```

### import 分区

按以下顺序分组，每组间空一行，组内按依赖层级排序：

```tsx
'use client';  // 如需

/*== 依赖导入 ==*/
import React from 'react';

/*== 组件导入 ==*/
import { Xxx } from '@/components/ui/xxx';

/*== 数据与配置 ==*/
import { something } from '@/lib/domain/xxx';

/*== 样式导入 ==*/
import styles from './foo-bar.module.css';

/*== 类型定义 ==*/
interface FooBarProps { ... }
```

短文件（一两行 import）可省略分区注释。

### Props 定义

- 使用 `interface`，就近定义在组件上方
- 每个字段带 `/*-- xxx --*/` 行内注释
- 不要用 `/** JSDoc */` 注释 props（除非是给外部消费的库 API）

```tsx
interface FooBarProps {
    /*-- 标题文字 --*/
    title: string;
    /*-- 是否禁用，默认 false --*/
    disabled?: boolean;
}
```

### 组件函数

- 使用 `function` 关键字，不用箭头函数
- 函数上方用 `/*== xxx ==*/` 一行描述
- 解构 props 带默认值

```tsx
/*== FooBar 组件描述 — 一句话说明 ==*/
export function FooBar({ title, disabled = false }: FooBarProps) {
    // ...
}
```

### JSX 内联注释

关键区块用 `{/*-- xxx --*/}` 注释标注：

```tsx
{/*-- 标题栏 --*/}
<div className={styles.header}>
    <h2>{title}</h2>
</div>
{/*-- 内容区 --*/}
<div className={styles.body}>{children}</div>
```

## 2. 条件渲染

### 三元 → Show 组件

`{condition ? <X /> : null}` 替换为 `<Show when={condition}><X /></Show>`：

```tsx
// ❌ 旧写法
{post.coverImage ? <img src={post.coverImage} /> : null}

// ✅ 新写法
<Show when={post.coverImage}>
    <img src={post.coverImage} />
</Show>
```

`<Show>` 支持 `fallback` 属性：

```tsx
<Show when={items.length > 0} fallback={<EmptyState />}>
    <ItemList items={items} />
</Show>
```

> `Show` 组件位于 `src/components/ui/show.tsx`。

## 3. CSS Module 优化

### 分区注释

每个样式块用 `/*-- xxx --*/` 标注：

```css
/*== FooBar 组件描述 — 一句话 ==*/

/*-- 容器 --*/
.root {
    display: flex;
    gap: var(--space-4);
}

/*-- 标题 --*/
.title {
    font-size: var(--text-xl);
    color: var(--foreground);
}
```

### Token 化

| 原始值 | 替换为 | 来源 |
|--------|--------|------|
| `#xxx` / 颜色值 | `var(--primary)` / `var(--foreground)` / `var(--muted)` 等 | `theme.css` |
| `12px` / `16px` / `24px` | `var(--space-3)` / `var(--space-4)` / `var(--space-5)` | `tokens.css` |
| `14px` / `16px` / `18px` 字号 | `var(--text-sm)` / `var(--text-base)` / `var(--text-lg)` | `tokens.css` |
| 组件内字号 | `var(--comp-text-sm)` / `var(--comp-text-base)` | `tokens.css` |
| `border-radius` | `var(--radius)` | `tokens.css` |
| `0.75rem` / `1.25rem` | `var(--space-3)` / `var(--space-5)` | `tokens.css` |
| `rgba(0,0,0,0.2)` | `var(--overlay)` | `theme.css` |
| `line-height: 1.625` | `var(--leading-relaxed)` | `tokens.css` |
| `font-weight: 400` | `var(--weight-normal)` | `tokens.css` |

### 不强行 token 化的例外

- 像素级精度：热力图单元格 `11px`、分隔线 `1px`、icon 占位 `3px`
- 响应式值：`clamp()`、`100vh`、媒体查询断点
- 渐变端点：`linear-gradient(... 2rem)` 中的长度
- `border-width`：用裸 `1px` / `2px`

判定原则：token 表达的是"刻度"，不是"所有 rem/px"。答不上"这个值属于哪个刻度"就保留原值。

## 4. 组件注册

优化完成后，需同步更新：

### Showcase 注册

1. 在 `src/showcase/demos.tsx` 添加 Demo 组件
2. 在 `src/showcase/registry.ts` 登记条目

Demo 示例要求：
- 覆盖组件的主要尺寸/变体/状态
- 用 `/*-- xxx --*/` 标注每个展示区域
- 附带代码块展示使用方式
- 交互型组件展示完整交互流程（如 loading 态）

### 文档路径更新

如果组件路径变更，同步更新：
- `src/docs/features/review-checklist.md`
- `src/docs/AGENTS.md`
- 相关功能文档（`src/docs/features/*.md`）
