## 2. 公开博客

### 2.1 首页

- Hero 区展示站点品牌、简介和页面锚点入口。
- 个人信息区展示头像、简介、社交操作和最近 27 周 Git 提交热力图。
- 文章区展示最近更新的 3 篇已发布文章。
- 项目区展示当前静态项目卡片。
- RSS 按钮复制 `/feed.xml` 订阅地址。

### 2.2 文章列表

- 使用分类单选和标签多选筛选文章。
- 筛选、标签和页码均写入 URL；移除筛选时返回第一页。
- 每页展示 10 篇文章，并提供上一页、页码和下一页导航。
- 桌面端使用列表加筛选侧栏，窄屏使用筛选弹窗。
- 只展示 `published` 状态文章。

### 2.3 文章详情

- 展示三级面包屑、文章头部、Markdown 正文、标签和页尾操作。
- 支持 GFM 表格、任务列表、代码高亮、站内图片和代码块复制。
- 仅在更新时间与发布时间不同时展示“最后更新于”。
- 按相同标签推荐最多 3 篇其他已发布文章。
- 生成文章、面包屑等 JSON-LD 数据。

### 2.4 公开分发与兼容

- `/feed.xml` 输出 RSS。
- `/sitemap.xml`、`/robots.txt` 由 App Router 元数据路由生成。
- `manifest.json`、favicon、Open Graph 和 Twitter Card 由根布局配置。
- 旧站路径通过集中映射执行永久重定向。

## 3. 导航页

### 3.1 搜索与书签

- 六种搜索引擎切换，回车在新窗口打开搜索结果。
- 书签支持普通链接和文件夹，提供 favicon、右键增删改和拖拽排序。
- 第一屏输入关键词后点击 AI 按钮，可跳转到第二屏并把关键词作为首条消息。

### 3.2 AI 对话

- 登录用户可选择可用 DeepSeek 模型并发起 SSE 流式对话。
- 支持多会话、新建、切换、删除和多轮上下文。
- Assistant 内容使用站内 Markdown 组件渲染。
- 模型偏好保存在浏览器本地，对话历史按登录状态使用数据库或本地存储。

### 3.3 备忘录与笔记

- 备忘录支持新增、勾选完成、删除和拖拽排序。
- 笔记支持新增、选择、Markdown 编辑、删除和拖拽排序。
- 游客保存在 localStorage；登录用户通过用户 API 保存到各自的 MySQL JSON 记录。

### 3.4 认证与同步

- 导航页内置登录/注册弹窗。
- 登录状态由 `use-auth.ts` 查询和维护。
- 用户登录后可将游客时期的本地数据批量同步到服务端。

## 4. 后台管理

### 4.1 后台壳层与登录

- `/admin/login` 不加载后台侧栏；已登录管理员直接进入概览页。
- `/admin/posts/:id` 使用独立全屏编辑器；其他后台页使用固定侧栏和主内容区。
- 侧栏菜单由集中配置驱动，支持当前分组自动展开、手动折叠、当前页标记和退出登录。

### 4.2 概览

- 展示文章总数、已发布数、草稿数、管理员数和普通用户数。
- 展示最近 5 篇文章并提供编辑入口。

### 4.3 文章管理与编辑

- 列表支持服务端首屏、关键词搜索、状态筛选、数据库分页、新建、删除和 ZIP 导出。
- 编辑器支持编辑、预览、分栏以及手动保存和 3000ms 延迟自动保存。
- 元数据包括标题、Slug、摘要、分类、标签、封面、封面替代文本和发布状态。
- 正文支持工具栏插入 Markdown，并支持选择、粘贴或拖拽图片后上传插入。
- 编辑预览直接复用博客详情 `ArticleDetail` 渲染链，显示真实面包屑、头部、正文和页尾。

### 4.4 分类标签与图片

- 分类和标签列表由服务端提供首屏数据，客户端支持创建、编辑和删除。
- 图片列表由服务端提供首屏数据，客户端支持上传、分页浏览、复制地址和删除。
- 服务器图片同步工具按差异把文件补充到本地 `public/uploads/`，不删除本地额外图片。

### 4.5 用户与设置

- 用户列表由服务端提供首屏数据，客户端支持分页搜索、创建、编辑用户名/邮箱/密码/角色/状态和删除。
- 设置页说明当前登录权限和项目约定。
- SEO 提交操作把首页、文章列表和已发布文章 URL 提交给已配置的 IndexNow 与百度端点。

## 5. 网站统计（观澜）

### 5.1 站点管理

- 创建统计站点并生成 8 位站点 ID。
- 编辑名称和域名，切换启用/暂停状态，执行软删除。
- 生成并复制包含 `data-site-id` 的异步嵌入脚本。

### 5.2 数据采集

- `public/script.js` 采集页面浏览、SPA 路由变化、会话开始、新访客和离开停留时长。
- 事件每 5 秒批量上报，离开页面时优先使用 `sendBeacon` 立即发送。
- `/api/collect` 接收公开事件，执行 CORS、站点校验、限流、字段约束、去重和批量入库。

### 5.3 统计概览与访问记录

- 在 7、30、90 天范围内查看 PV、UV、跳出率、平均停留和新访客比例。
- 查看趋势、热门页面、来源、设备、浏览器、操作系统、国家/地区、语言、入口和出口页面。
- 查看原始访问记录并分页浏览。
- 查询前按需补齐日聚合，按现有规则清理 90 天前原始事件。
