## 1. 产品定位

知简是一个个人使用的 Next.js 全栈站点，当前由五部分组成：

1. 面向访客的个人技术博客。
2. 面向个人效率场景的导航页。
3. 面向管理员的内容、用户、图片和站点数据后台。
4. 面向被统计站点的轻量访问数据采集与分析能力（观澜）。
5. 面向项目维护者的组件预览、仓库文档和质量校验能力。

## 2. 用户与权限

| 角色               | 当前能力                                                   | 权限边界                               |
| ------------------ | ---------------------------------------------------------- | -------------------------------------- |
| 游客               | 浏览博客、使用导航页本地数据、登录或注册                   | 不能调用用户和后台受保护接口           |
| 注册用户（`user`） | 使用导航页，并将书签、备忘录、笔记和 AI 对话同步到个人账户 | 不能进入后台或调用 admin 接口          |
| 管理员（`admin`）  | 使用全部后台功能，并具备普通用户登录能力                   | 后台页面和 admin API 均需管理员会话    |
| 外部访客浏览器     | 通过嵌入脚本向观澜上报访问事件                             | 只能调用公开采集端点，不能读取统计数据 |

## 4. 非功能需求

### 4.1 安全

- 用户密码使用 bcrypt 12 轮哈希，不保存明文密码。
- Session Token 使用 `ADMIN_SESSION_SECRET` 进行 HMAC-SHA256 签名，并使用时序安全比较校验签名。
- 生产环境 Session Cookie 启用 `secure`，所有环境均使用 `httpOnly`、`sameSite=lax` 和根路径作用域。
- 数据库查询使用参数化占位符；API 在路由边界校验不可信输入。
- 公开采集接口不返回统计数据，并执行按站点限流和字段约束。

### 4.2 一致性与可维护性

- 路由页面负责编排，业务组件放在 `components/`，业务数据访问放在 `lib/domain/`，基础设施放在 `lib/core/`。
- 前台博客与后台文章预览必须复用同一文章详情渲染链，避免展示差异。
- API 使用统一响应结构和鉴权包装器；路由文件不直接编写 SQL。
- 全站使用 TypeScript 严格模式、CSS Modules 和现有 CSS Token，不引入平行样式体系。

### 4.3 性能与数据口径

- 博客使用 Server Component 获取公开数据；同一请求内通过 React `cache()` 避免元数据和页面重复查询。
- 后台文章列表首屏在服务端获取，筛选和分页在数据库完成，不向客户端传输全文字段。
- 首页只读取 3 篇文章，文章详情相关文章查询限制数量。
- 统计查询优先使用日聚合表，聚合和查询统一按当前 UTC 转换规则处理日期范围。
- 内容图片优先使用项目现有的 Next.js 图片封装；埋点脚本异步加载并批量上报。

### 4.4 文档与质量门禁

- 代码实现是当前行为的最终事实来源；文档不得描述尚未实现的能力。
- 新增、删除或调整功能、路由、接口、数据模型、架构边界和校验流程时，必须在同一提交中更新对应文档。
- 提交前运行 `npm run check`；涉及行为逻辑时补充并运行相应测试。
