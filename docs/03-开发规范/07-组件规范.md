# 07 · 组件规范

- 文档状态：开发规范
- 适用范围：当前项目
- 最后核对：2026-07-18
- 维护范围：UI、站点和业务组件的分层、组织与注册规则
- 关联代码：`src/components/`、`src/showcase/`

## 三层组件

| 层       | 目录                           | 定位                         | 依赖                        |
| -------- | ------------------------------ | ---------------------------- | --------------------------- |
| 原子     | `components/ui/`               | 无业务语义、跨站可复用       | 最多依赖 `@/lib/core/utils` |
| 前台展示 | `components/site/`             | 带站点语义，但不绑定单一路由 | `ui` + 只读 domain          |
| 业务复合 | `components/modules/<domain>/` | 由多个原子组件拼出的业务块   | `ui` + domain               |

判断标准：

- 组件统一放 `components/`，`app/` 只负责路由编排和组件消费。
- 无业务语义的原子组件放 `components/ui/`。
- 有站点语义但不绑单一页面的放 `components/site/`。
- 按功能域组合的业务块放 `components/modules/<域>/`。

## 页面与客户端边界

后台管理页优先采用以下结构：

```text
page.tsx（Server Component）
  ├─ metadata / 路由参数
  ├─ AdminPageHeader 等静态页面结构
  ├─ 首屏数据读取与必要字段裁剪
  └─ 业务 Client Component
       ├─ 动态数量和操作工具栏
       ├─ 筛选、分页和即时交互
       └─ Dialog / ConfirmDialog
```

规则：

- `page.tsx` 负责路由级编排，不把 Hooks、浏览器 API 或完整 CRUD 状态搬进页面。
- 静态页面头部不要放在整页 Client Component 中；动态数量和依赖客户端状态的按钮放在业务工具栏。
- 服务端已提供首屏数据时，客户端首次挂载不得重复请求；筛选、分页或操作后再调用 API。
- 只向 Client Component 传递实际使用且可序列化的字段，不透传数据库完整记录。
- 不为连接 Server Page 与 Client Component 再创建只转发 props 的包装组件。

## 文件组织

新增组件采用平铺方式，与同目录其他组件保持一致：

```text
components/ui/ghost-button.tsx
components/ui/ghost-button.module.css
```

组件过多时可按子目录分组。文件名 kebab-case，导出名 PascalCase，样式文件与组件同名。

## 组件写法

- 统一函数组件
- props 用 `interface`
- 副作用逻辑放到 hook 或更上层容器
- 样式优先走 CSS Module，不在 JSX 里堆 `style={{ ... }}`

示例：

```tsx
import { cn } from '@/lib/core/utils';
import styles from './ghost-button.module.css';

interface GhostButtonProps {
    children: React.ReactNode;
    variant?: 'default' | 'primary';
}

export function GhostButton({ children, variant = 'default' }: GhostButtonProps) {
    return <button className={cn(styles.button, styles[variant])}>{children}</button>;
}
```

注意：当前项目里统一使用的是 `cn(...)` helper，不写 `cx(...)`。

## 图标

图标集中在 `components/ui/icons.tsx`。当前策略是手写 SVG，不引图标库。

要求：

- 命名统一
- 视觉风格统一
- 能被 `/admin/showcase/icons` 直接预览

## showcase

组件展示索引放在 `src/showcase/registry.ts`，手写登记：

```ts
export interface ShowcaseEntry {
    name: string;
    description: string;
    module: 'ui' | 'site' | 'nav' | 'blog' | 'admin';
    source: string;
    examples: {
        label: string;
        Component: ComponentType;
        props?: Record<string, unknown>;
    }[];
}
```

规则：

- 新增可复用组件时，顺手补 registry
- 示例 props 要能真实渲染，不放占位假数据
- 不做自动反射

## 去重规则

出现两个页面写了结构明显相似的组件时：

1. 先看有没有现成组件可复用
2. 再决定是否抽到 `components/`
3. 抽完后删掉旧重复实现

不要让“记忆复用”替代“可见复用”。

## 禁止项

- 禁止组件内写 SQL、直接访问数据库
- 禁止 `ui` 反向依赖 `site` 或 `modules`
- 禁止 props 透传过深；超过两层就考虑提升状态或改结构
- 禁止样式硬编码，遵循[样式规范](./04-样式规范.md)
