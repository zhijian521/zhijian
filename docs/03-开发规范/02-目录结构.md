## 目录职责

### app/

只放路由、页面、layout。**业务逻辑和组件不进 app**——调 `lib/domain`，组件从 `components/` 导入。

- `app/*/_hooks/`:仅页面内 hook,下划线前缀=不参与路由。
- 不再使用 `app/*/_components/`，所有组件统一放 `components/modules/<域>/`。

### components/

| 目录            | 放什么              | 判定标准                                                           |
| --------------- | ------------------- | ------------------------------------------------------------------ |
| `ui/`           | 原子组件,无业务语义 | `Button` `Dialog` `TextInput`,换站点也能用                         |
| `site/`         | 前台展示组件        | 文章卡、页头页脚,有站点语义但不绑单一页面                          |
| `modules/<域>/` | 业务复合组件        | 把 `ui` 组合成业务区块,按功能域组织(如 `home/`、`blog/`、`admin/`) |

所有组件统一收归 `components/` 管理，`app/` 只负责路由编排和组件消费。

### lib/

| 目录      | 放什么                                                  | 依赖方向                        |
| --------- | ------------------------------------------------------- | ------------------------------- |
| `core/`   | db 连接、auth/session、api-response、http-client、utils | 不依赖 domain                   |
| `domain/` | posts、taxonomy、nav、uploads、analytics 等业务数据层   | 依赖 core,不依赖 app/components |

**单向依赖**:`app → domain → core`。core 永不反向 import domain。

### showcase / docs

- `src/showcase/` 使用手写 Registry 驱动后台组件和图标预览。
- 根目录 `docs/` 只存 Markdown，不参与应用运行时；分类与维护规则见[文档与校验](./08-文档与校验.md)。

## 文件粒度

- 组件采用平铺方式：`模块.tsx` + `模块.module.css` 直接放在所属目录下。同一目录下组件过多时再考虑分组子目录。
- 一个 domain 一个文件:`domain/posts.ts`,内部聚合该域所有数据访问。
- 不要为「将来可能拆」预留空目录。

## 路径别名

`tsconfig.json` 配 `@/*` 指向 `src/*`。import 一律用 `@/lib/core/db`,不用相对路径跨目录。
