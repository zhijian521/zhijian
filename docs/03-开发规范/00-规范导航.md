## 三条总要求

1. 代码要简洁，避免样板和为未来预留的空壳。
2. 结构要清楚，目录职责、依赖方向和组件层级保持稳定。
3. 规范要可执行，文档不能只写原则，必须落到脚本或检查清单。

## 冲突时怎么判定

按以下优先级处理，越靠前越优先：

1. 仓库内可执行事实：脚本、配置和当前实现。
2. 更具体、更贴近当前主题的专项规范。
3. 更泛的总则和导航说明。

如果规范与 `package.json`、校验脚本或真实代码短暂漂移，先以可执行事实为准，并在同一变更中修正文档。

## 统一完成标准

日常开发统一以 `npm run check` 作为完成前校验入口，当前包含：

- `npm run lint`
- `npm run typecheck`
- `npm run docs:check`

涉及行为逻辑时运行 `npm test`。格式化由 `npm run format` / `npm run format:check` 辅助，当前尚未纳入统一 `check`。
