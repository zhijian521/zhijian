## 环境变量

复制 `.env.example` 为 `.env.local`，根据实际环境修改：

```env
# 站点地址，用于 OG 标签、sitemap、绝对 URL 生成
NEXT_PUBLIC_SITE_URL=https://yuwb.dev

# 监控脚本分发地址（不配置时回退到 NEXT_PUBLIC_SITE_URL）
NEXT_PUBLIC_SCRIPT_URL=https://yuwb.dev

# MySQL 连接串
DATABASE_URL=mysql://username:password@host:3306/zhijian

# 后台管理员账号
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your-password

# 后台登录态签名密钥，随机长字符串
ADMIN_SESSION_SECRET=replace-with-a-long-random-string
```

## 构建与启动

```bash
npm run build
npm run start
```

默认监听 3000 端口，通过 Nginx 反向代理到 80/443。

## Nginx 配置

### 不要给页面加缓存

博客内容是动态的（文章列表、后台数据都从数据库实时读取），如果给 HTML 页面加了缓存，用户看到的可能是旧内容，后台改了文章前台不更新。

```nginx
# ❌ 不要这样写
location / {
    proxy_cache my_cache;
    proxy_pass http://127.0.0.1:3000;
}

# ✅ 正确写法：不缓存页面，直接代理
location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection 'upgrade';
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

Next.js 自身的静态资源（`/_next/static/`）文件名带 hash，浏览器天然缓存，不需要 Nginx 额外处理。

### 反向代理 uploads 目录

后台文章封面上传的图片存储在 `public/uploads/` 目录。Next.js 生产模式（`next start`）只服务构建时存在的静态文件，运行时写入的文件访问会 404。需要 Nginx 直接代理这个目录：

```nginx
location /uploads/ {
    alias /项目绝对路径/public/uploads/;
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```

- `alias` 指向项目 `public/uploads/` 的绝对路径
- 上传的图片不会变（文件名是 hash），可以放心加长缓存
- 这条规则要放在 `location /` 之前，Nginx 优先匹配更长路径
