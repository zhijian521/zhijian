## 1. 架构概览

知简是一个基于 Next.js App Router 的单体全栈应用。页面、API、静态资源和维护工具位于同一仓库，业务数据主要存储在 MySQL。

```text
浏览器
  ├─ 公开页面 / 导航页 / 后台页面
  ├─ public/script.js（嵌入到被统计站点）
  └─ localStorage / sessionStorage（导航游客数据、统计访客标识）
           │
           ▼
Next.js 15 App Router
  ├─ Server Components：页面编排、首屏取数、元数据
  ├─ Client Components：表单、筛选、编辑器、图表和即时交互
  ├─ Route Handlers：公开 API、用户 API、后台 API、RSS
  └─ Middleware：注入 x-current-path，帮助服务端布局选择壳层
           │
           ▼
lib/domain（业务与数据访问）
           │
           ▼
lib/core（数据库、认证、响应、HTTP、配置、基础工具）
           │
           ├─ MySQL
           ├─ DeepSeek API
           ├─ GitHub API
           ├─ IndexNow / 百度提交端点
           └─ ip2region 本地数据
```

## 3. 目录与职责

```text
src/
├─ app/                    路由、layout、页面和 Route Handler
├─ components/
│  ├─ ui/                 无业务语义的原子组件
│  ├─ site/               前台共享展示与文章渲染组件
│  └─ modules/<domain>/   按业务域组织的复合组件
├─ lib/
│  ├─ core/               基础设施和跨域底层能力
│  └─ domain/             业务类型、规则和数据访问
├─ hooks/                 跨页面客户端 Hook
└─ showcase/              组件展示手写 Registry

public/                   静态资源、上传图片、PWA 资源、统计脚本
sql/                      数据库初始化和管理员 seed
scripts/                  文档校验与图片同步等维护脚本
docs/                     产品、技术和开发规范 Markdown
```

### 3.1 依赖方向

```text
app ───────────────┐
                   ├─> domain ─> core
components ────────┘

app ─> components/ui | components/site | components/modules
components/modules ─> components/ui | components/site
```

强制边界：

- `core` 不依赖 `domain`、`components` 或 `app`。
- `domain` 可以依赖 `core`，但不依赖 Next.js 的请求和响应对象。
- `app/api/**/route.ts` 负责协议、鉴权和输入输出编排，SQL 位于 `lib/domain/` 或当前既有认证基础设施中。
- `app/**/page.tsx` 负责路由级取数和组合，不承载可复用业务组件。
- 跨层导入使用 `@/` 路径别名。

## 4. 渲染与壳层

### 4.1 根布局

`src/app/layout.tsx` 设置全站元数据并渲染 `AppFrame`。`AppFrame` 根据路径把公开页面交给 `PublicChrome`，后台页面交给后台布局自身处理。

`src/middleware.ts` 为应用请求注入 `x-current-path`，供服务端布局区分当前路由，不承担业务鉴权。

### 4.2 公开页面

- 首页、博客列表和详情是 Server Component，并使用 `force-dynamic` 获取当前数据库数据。
- 文章列表把服务端计算好的筛选项、分页链接和文章数据交给轻量 Client Component 处理路由切换状态。
- 文章详情由 `ArticleDetail` 组合 `Breadcrumb`、`ArticleView`、`MarkdownArticle` 和页尾。
- Markdown 渲染、内容图片和代码块是前台与编辑预览共享的真实组件链。

### 4.3 后台页面

`src/app/admin/layout.tsx` 有三种分发模式：

| 路由               | 壳层                        | 鉴权                 |
| ------------------ | --------------------------- | -------------------- |
| `/admin/login`     | 登录页裸渲染                | 页面自行处理已有会话 |
| `/admin/posts/:id` | 全屏编辑器 + Toast          | `requireAdmin()`     |
| 其他 `/admin/*`    | 固定侧栏 + 主内容区 + Toast | `requireAdmin()`     |

后台通常采用以下组合：

```text
page.tsx（Server Component）
  ├─ 元数据 / 路由参数 / AdminPageHeader
  ├─ 获取并裁剪首屏数据
  └─ 业务 Client Component（只接收实际使用字段）
       ├─ 交互状态
       ├─ 后续 API 请求
       └─ 工具栏 / 列表 / 图表 / 弹窗
```

文章、统计站点、分类标签、图片和用户管理均采用该边界：静态页面头部留在服务端，动态数量和操作入口放在客户端工具栏；首屏数据由路由页提供，避免挂载后重复请求。

## 5. API 架构

API Route 按权限和业务域分组：

| 路径           | 权限            | 主要用途                         |
| -------------- | --------------- | -------------------------------- |
| `/api/posts`   | 公开            | 已发布文章列表                   |
| `/api/auth/*`  | 公开 / 当前会话 | 注册、登录、登出、当前用户       |
| `/api/nav/*`   | 用户            | 个人导航数据读取、保存和同步     |
| `/api/ai/*`    | 用户            | 模型列表与 DeepSeek 流式对话     |
| `/api/admin/*` | 管理员          | 内容、用户、图片、SEO 和统计管理 |
| `/api/collect` | 公开采集        | 接收被统计站点事件               |
| `/api/favicon` | 公开            | 导航书签 favicon 代理            |

统一约定：

- 后台 Route Handler 使用 `withAdmin()`，用户 Route Handler 使用 `withUser()`。
- JSON 接口使用 `success()` / `fail()` 和统一业务码。
- 列表接口返回数据列表和总数；分页参数由 `lib/core/pagination.ts` 统一解析。
- 动态路由参数在 Next.js 15 中按 Promise 读取。
- Route Handler 只处理 HTTP 边界，业务查询调用 domain 函数。

所有当前 Route Handler 都登记在 `docs/02-技术文档/03-接口文档/`，`docs:check` 对真实路由和 Markdown 二级标题做双向校验。

## 6. 认证架构

```text
注册 / 登录
  └─ bcrypt 校验或生成密码哈希
       └─ HMAC-SHA256 Session Token
            └─ zhijian_session HttpOnly Cookie（7 天）
                 ├─ Server Page：getSessionFromCookies / requireAdmin
                 └─ API：getSessionFromRequest -> withUser / withAdmin
```

- Token 内容为用户 ID、用户名、角色和过期时间，并由 `ADMIN_SESSION_SECRET` 签名。
- `user` 和 `admin` 使用同一会话格式；包装器根据角色执行额外判断。
- 密码使用 bcrypt 12 轮哈希。
- 登录页可选择只在 localStorage 记住用户名，不保存密码或 Session Token。

## 7. 数据架构

### 7.1 数据库连接

- `lib/core/db.ts` 创建模块级 MySQL 连接池并在进程内复用。
- 连接时区设置为 `+08:00`，字符集使用 `utf8mb4`。
- 未配置 `DATABASE_URL` 时 `getDb()` 返回 `null`；公开文章数据层可使用内置示例数据回退。

### 7.2 当前数据表

| 表组 | 表                                                                                    | 用途                             |
| ---- | ------------------------------------------------------------------------------------- | -------------------------------- |
| 用户 | `zhijian_users`                                                                       | 账号、角色、状态和密码哈希       |
| 博客 | `zhijian_blog_posts`                                                                  | 文章正文、元数据、状态和发布时间 |
| 博客 | `zhijian_blog_categories`、`zhijian_blog_tags`                                        | 分类与标签                       |
| 博客 | `zhijian_blog_uploads`                                                                | 上传图片元数据                   |
| 统计 | `zhijian_track_sites`                                                                 | 被统计站点及状态                 |
| 统计 | `zhijian_track_events`                                                                | 原始 pageview / leave 等事件     |
| 统计 | `zhijian_track_daily`                                                                 | summary / page / dim 日聚合行    |
| 导航 | `zhijian_nav_bookmarks`、`zhijian_nav_todos`、`zhijian_nav_notes`、`zhijian_nav_chat` | 每用户一条 JSON 数据             |

完整字段、索引和外键以 `sql/init.sql` 为准。

### 7.3 文件数据

- 上传图片保存到 `public/uploads/YYYY/MM/`，元数据写入 `zhijian_blog_uploads`。
- 静态站点图片、PWA 图标和 GeoIP 数据位于 `public/`。
- 游客导航数据、模型偏好和统计访客标识保存在浏览器存储中，键由相应 domain 或脚本维护。

## 8. 主要业务数据流

### 8.1 博客读取

```text
page.tsx / generateMetadata
  -> React cache 包装的查询
  -> lib/domain/posts.ts
  -> MySQL
  -> Post 业务对象
  -> site/modules 组件渲染
```

公开查询只返回已发布文章；后台查询可读取草稿。标签名称通过批量查询补齐，避免按文章逐条查询。

### 8.2 文章编辑与预览

```text
编辑器本地表单状态
  ├─ 3000ms 延迟 -> 串行 PATCH /api/admin/posts/:id -> domain/posts -> MySQL
  └─ 组装完整 Post -> ArticleDetail -> 与博客详情一致的实时预览
```

编辑器使用版本标识防止较旧保存响应覆盖较新的本地状态。上传图片先写入文件系统和上传表，再把 Markdown 图片语法插入正文。

### 8.3 导航双端存储

```text
Nav Client Component
  -> nav-storage.ts
       ├─ 游客 -> localStorage
       └─ 已登录 -> /api/nav/* -> nav-db.ts -> MySQL JSON
```

登录后的批量同步通过 `/api/nav/sync` 把本地书签、备忘录和笔记写入当前用户记录。AI 对话历史使用独立接口，避免首屏全量读取。

### 8.4 统计采集与查询

```text
被统计站点 public/script.js
  -> 批量 POST /api/collect
  -> 校验 / 限流 / 去重 / GeoIP / UA
  -> zhijian_track_events
  -> ensureAggregated / aggregateDaily
  -> zhijian_track_daily
  -> admin analytics API
  -> AnalyticsDashboard / Overview / Visits
```

日期查询将本地日期转换为 UTC 范围，原始事件和聚合数据使用同一口径。统计容器使用请求 ID 防止较慢响应覆盖当前筛选结果。

### 8.5 文章导出与图片同步

- 导出 API 读取单篇或全部文章，解析正文中的站内上传图片，并通过 archiver 流式生成 ZIP。
- 图片同步脚本读取服务器文件清单，仅下载本地缺少或内容不同的文件，不删除 `public/uploads/` 中的本地额外文件。

## 9. 样式与组件架构

样式分三层：

```text
tokens.css（间距、字号等基础 token）
  -> theme.css（文人书斋颜色、字体和语义变量）
     -> globals.css / admin/globals.css（重置和少量全局规则）
        -> *.module.css（组件局部样式）
```

组件分层：

- `components/ui`：Button、Dialog、Select、Table、Pagination、Tag、Toast 等无业务语义组件。
- `components/site`：公开壳层、站点头部、文章视图、Markdown、面包屑等站点级共享组件。
- `components/modules`：首页、博客列表和后台各业务组件。
- `src/app/nav/_components`：当前导航页路由内专用组件，尚未作为跨域组件复用。

博客与后台采用朱砂红、宣纸米白、衬线标题和零圆角；导航页复用主题颜色，但保留自己的小圆角变量。

## 11. 架构变更规则

出现以下变化时，代码和文档必须在同一提交中更新：

- 产品范围或角色能力变化：更新需求总览、功能总览和对应专项文档。
- 页面、路由或交互变化：更新功能总览和对应专项文档。
- 分层、目录、依赖或运行时数据流变化：更新本架构文档和对应开发规范。
- API 新增、删除或契约变化：更新 route 顶部注释、对应接口 Markdown 和所属功能文档。
- 数据表、字段或数据口径变化：更新 `sql/init.sql`、所属功能文档；影响整体模型时同时更新本架构文档。
- 校验命令或脚本变化：更新 `docs/03-开发规范/08-文档与校验.md`、规范导航和根 README。
