# 公开与认证接口

- 文档状态：当前实现基线
- 适用范围：当前项目
- 最后核对：2026-07-21
- 维护范围：公开博客、认证和 Favicon 代理接口
- 关联代码：`src/app/api/posts/`、`src/app/api/auth/`、`src/app/api/favicon/`

本文档记录当前 6 个接口。除特别说明外，“成功数据”指统一响应结构中的 `data` 字段。

## /api/posts · 公开文章列表

- 鉴权：公开
- 路由文件：`src/app/api/posts/route.ts`

### GET · 分页获取已发布文章

- Query：`page?: number`、`pageSize?: number`，由 `lib/core/pagination.ts` 统一解析。
- 默认值：`page=1`、`pageSize=10`；`pageSize` 最大为 `100`。
- 成功数据：`{ data: Post[], total: number }`，分页在 SQL 层完成（`COUNT(*)` 与 `LIMIT/OFFSET` 并行查询）。
- 列表字段：不返回正文内容（`content` 恒为空字符串）。
- 渲染策略：强制动态读取当前数据库内容。

## /api/auth/login · 用户登录

- 鉴权：公开
- 路由文件：`src/app/api/auth/login/route.ts`

### POST · 使用用户名和密码登录

- JSON Body：`username: string`、`password: string`，均必填。
- 成功状态：`200`。
- 成功数据：`{ user: { id, username, email, role } }`。
- 副作用：设置 7 天有效的 `zhijian_session` HttpOnly Cookie。
- 限流：按 IP + 用户名（截断前 50 字符）5 次/分钟，超限返回 `429`（`RATE_LIMITED`）。IP 由 `getClientIp()` 提取（`x-real-ip` 优先，否则 `x-forwarded-for` 链最右侧非空值，防伪造首值）；IP 无法识别时降级为仅按用户名限流并 `console.warn` 记录。

## /api/auth/logout · 用户登出

- 鉴权：公开
- 路由文件：`src/app/api/auth/logout/route.ts`

### POST · 清除当前登录态

- 请求：无。
- 成功数据：`null`。
- 副作用：将 `zhijian_session` Cookie 的 `maxAge` 设为 `0`。

## /api/auth/me · 当前用户信息

- 鉴权：公开入口，接口内部校验 Session；未登录返回 `401`。
- 路由文件：`src/app/api/auth/me/route.ts`

### GET · 获取当前会话用户

- 请求：浏览器自动携带 `zhijian_session` Cookie。
- 成功数据：`{ user: { id, username, email, role } }`。
- 失败：Session 无效、用户不存在或账号被禁用时返回 `401`。

## /api/auth/register · 用户注册

- 鉴权：公开
- 路由文件：`src/app/api/auth/register/route.ts`

### POST · 注册普通用户

- JSON Body：`username: string`、`email: string`、`password: string`，均必填。
- 成功状态：`201`。
- 成功数据：`{ user: { id, username, email, role } }`，其中 `role` 固定为 `user`。
- 冲突：用户名或邮箱重复时返回 `409`。
- 限流：按 IP 5 次/分钟，超限返回 `429`（`RATE_LIMITED`）。IP 提取策略同登录接口（`getClientIp()`，XFF 链尾优先于可伪造首值）。

## /api/favicon · Favicon 代理

- 鉴权：公开
- 路由文件：`src/app/api/favicon/route.ts`

### GET · 获取目标域名的站点图标

- Query：`domain: string`，必填，只接受合法域名，不包含协议和路径。
- 成功响应：图片二进制，携带 `Content-Type` 和 30 天浏览器缓存头。
- 缓存：服务端内存最多保存 500 项，每项 30 天。
- SSRF 防护：抓取统一经过 `src/lib/core/ssrf-guard.ts` 的 `fetchWithSsrfGuard`——仅允许 https 协议；主机名为域名时解析 DNS，全部解析结果必须为公网地址（拒绝私网、回环、链路本地、保留段，含 `::ffff:` IPv4 映射地址；并整段拒绝内嵌 IPv4 的过渡机制前缀 NAT64 `64:ff9b::/96` 与 `64:ff9b:1::/48`、Teredo `2001::/32`、6to4 `2002::/16`，防止映射到云元数据等内网地址）；重定向手动跟随，每一跳重新校验。
- 读取上限：首页 HTML 流式读取，超过 512KB 即截断；favicon 图片流式读取限 2MB，超限视为抓取失败走下方降级，防止无鉴权端点被大响应体耗尽内存。
- 降级：无法抓取目标图标时返回 `302`，重定向到 Google Favicon 服务。
- 失败：域名不合法时返回 `400 { error: 'invalid domain' }`，不使用统一响应结构。
