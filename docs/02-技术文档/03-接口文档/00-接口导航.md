## 鉴权级别

| 标记       | 说明                                               |
| ---------- | -------------------------------------------------- |
| 公开       | 不要求登录；接口仍需执行对应输入校验和安全限制     |
| 用户登录   | 使用 `withUser()`，允许有效 `user` 或 `admin` 会话 |
| 管理员登录 | 使用 `withAdmin()`，仅允许 `admin` 会话            |

## 维护规则

- 新增、删除或调整 API 时，同一提交内更新对应接口 Markdown 和 Route Handler 顶部注释。
- 接口路径、方法、鉴权、参数或响应发生变化时，必须更新对应条目。
- 接口路径使用 `## /api/... · 说明` 二级标题，每个 HTTP 方法使用 `### GET · 说明` 三级标题独立描述。
- `npm run docs:check` 双向核对磁盘 Route Handler 与 Markdown 中登记的接口路径和 HTTP 方法。
- 接口实际行为与文档冲突时，以当前 Route Handler 为事实来源，并立即修正文档。
