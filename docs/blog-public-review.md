# 博客公开页面审查与处理方案

## 文档目标

本文档用于整理当前对外开放博客页面的审查结果，范围包括：

- 首页 `/`
- 博客列表页 `/blog`
- 博客详情页 `/blog/[slug]`

重点关注以下几类问题：

- 是否存在不合理实现
- 是否有遗留未删除代码
- 是否有可以简化的实现
- 是否有更合适、更稳定的替代方案
- 是否存在潜在 Bug 或线上风险

---

## 当前结论

当前博客公开页面的大部分高优先级问题已经完成，已落地内容包括：

- 移动端公共导航可用性修复
- `/blog` 服务端分页、服务端分类/标签筛选
- `/blog` 页去除本地 `Suspense`，列表组件收敛为纯渲染
- 列表页筛选结果的 `noindex, follow` 策略
- 详情页文章查询结果复用
- 详情页独立 loading 路由补齐并与站点全局加载样式对齐
- 日期解析与时区统一
- 公开页链接预取策略收敛
- 详情页结构化数据增强
- 公开博客链路图片渲染策略统一

当前真正还值得继续优化的重点，已经收敛为：

- 列表页参数职责是否继续向更纯服务端主导收口
- 详情页标签查询是否继续专项优化
- 文档与代码状态保持同步

## 建议验收方式

### 列表页相关

- 直接访问 `/blog?page=2` 能返回正确 HTML 内容
- 查看页面源码时，标题、描述、canonical、列表内容一致
- 带 `category` / `tags` 的筛选页输出 `noindex, follow`

### 详情页相关

- 单篇文章访问时数据库查询次数下降
- metadata 与页面主体使用同一份文章数据
- 结构化数据中的图片、publisher、阅读时长输出正确

### 时间相关

- 首页、列表页、详情页、sitemap 日期一致
- 本地与服务器显示结果一致

### 性能相关

- 首页进入列表、列表进入详情的体感更顺滑

---

## 最终建议

如果继续做最后一轮清理，优先建议：

1. 决定是否还需要继续收口列表页客户端 URL 逻辑
2. 如无明确性能瓶颈，详情页标签查询可以暂时保持现状
3. 保持文档和代码状态一致
